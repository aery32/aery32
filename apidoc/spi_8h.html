void aery_spi_init_master ( volatile avr32_spi_t *  pspi)

Init SPI as a master

Parameters:
pspiPointer to the SPI peripheral which to init
CS multiplexer
If you are using multiplexed chip selects, enable cs multiplexing by bitbanging PCSDEC bit in SPI MR register after initialization:
aery_spi_init_master(&AVR32_SPI0);
AVR32_SPI0.MR.pcsdec = 1;
Examples:
displays/hd44780.c, and spi.c.

uint8_t  npcs,
enum Spi_mode  mode,
uint8_t  bits 
)

Setups the SPI mode and the shift register width of the Numeric Processor Chip Select (NPCS, same as slave/chip select)

Parameters:
pspiPointer to the SPI peripheral
npcsChip Select: 0-3
modeSPI mode
bitsWidth of SPI shift register: 8-16 bits
Making SPI SCK faster
Chip select baudrate is hard coded to MCK/255. To make it faster you can bitbang the SCRB bit in CSRX register.
aery_spi_setup_npcs(&AVR32_SPI0, 0, SPI_MODE1, 16);
AVR32_SPI0.CSR0.scbr = 32; // baudrate is now MCK/32
Note:
Every chip select line can have different mode and size of shift register.
Examples:
displays/hd44780.c, and spi.c.

Writes and reads SPI bus

Parameters:
pspiPointer to the SPI peripheral which to use.
dataBinary word to be send.
npcsChip select line number. Can also be number of cs multiplexer. Use dummy cs number, if you like to use some other gpio pin through bitbanging.
islastIs this the last transmit? If no, set 0 to leave chip select low.
Returns:
Received data bits
SPI read
When you only want to read from external device through SPI, ignore the sent data and use dummy bits instead, for example 0x00.
uint16_t rd; // read data
rd = aery_spi_transmit(&AVR32_SPI0, 0, 0, true);
Examples:
displays/hd44780.c, and spi.c.
